# RobustQuant 术语表

版本：v0.1  
状态：草案

这份术语表用于辅助后续讨论。它不会追求百科全书式完整，而是先解释本项目里最容易混淆的概念。

## 量化交易

量化交易是把投资逻辑写成可重复执行的规则或模型。它不等于一定赚钱，也不等于一定使用 AI。最朴素的量化策略可以只是“均线突破买入，跌破均线卖出”。

## 策略

策略是一套决定买什么、什么时候买、买多少、什么时候卖的规则。

在系统里，策略不应该直接调用券商 API。策略只产生交易意图，真实下单必须经过风控和订单管理。

## 因子

因子是用来描述股票某个特征的数值。例如：

- 动量因子：最近一段时间涨得多不多。
- 估值因子：市盈率、市净率是否便宜。
- 质量因子：ROE、毛利率、现金流是否健康。
- 情绪因子：新闻和公告是否偏正面。

因子本身不是策略。因子更像是给股票打标签或打分的原材料。

## 选股

选股回答“买哪些股票”。例如模型每天从全市场选出评分最高的 20 只股票。

选股不等于下单。下单还需要考虑交易时机、仓位、风控和账户可用资金。

## 候选股票池

候选股票池是 AI、规则或数据源先筛出来的一批“可能值得研究”的股票。候选不等于正式纳入策略，更不等于下单。候选股票需要记录来源、理由、主题归属和置信度，并经过人工确认后才能进入正式研究股票池。

## 关键词匹配风险

关键词匹配风险是指系统只因为新闻或公告里出现了某个词，就机械地匹配到同名股票或相似概念股票。例如“美国总统特朗普抵达天坛”不能用来选择名称里带“天坛”的股票建仓。消息面必须先做实体识别、标的关联和真实影响判断。

## 信号

信号是策略在某个时点产生的判断。例如：

- 买入信号。
- 卖出信号。
- 调仓信号。
- 暂停交易信号。

## 消息面

消息面指新闻、公告、宏观数据、政策表态、财报等会影响市场预期的信息。它和行情价格不同，通常是非结构化或半结构化数据，需要先解析成事件类型、时间、影响对象、情绪方向等结构化字段，才能进入量化研究。

## 宏观事件

宏观事件是会影响整体市场风险偏好、利率预期或经济预期的数据和政策事件。例如 CPI、非农就业、失业率、FOMC 利率决议、PMI、GDP 等。

在系统里，宏观事件不直接触发下单。它们会被转换成事件信号，作为因子、策略过滤器或风控上下文。

## 事件 Surprise

事件 Surprise 指公布值相对市场预期的偏离程度。例如市场预期 CPI 同比为 3.0%，实际公布为 3.4%，这就是高于预期的 surprise。

Surprise 的方向不能简单等同于涨跌方向。市场反应还要看当时的利率预期、资产价格位置、风险偏好和是否已经提前反应。

## 快速风险退出

快速风险退出是指重大消息或宏观事件发生后，系统在短时间内把事件信号注入策略和风控上下文，让策略可以及时降低风险暴露。例如 CPI 明显高于预期，同时半导体板块快速下跌，系统可以触发“暂停开仓、降低相关持仓”的防守型动作。

它不是消息直接下单。快速风险退出仍然必须经过策略、风控、OMS 和券商网关，并且只允许降低风险暴露，不允许自动加仓。

## 回测

回测是用历史数据模拟策略过去会怎么交易、结果如何。回测的作用是排除明显不靠谱的策略，但回测好不代表实盘一定好。

常见回测陷阱：

- 未来函数：用了当时不可能知道的数据。
- 幸存者偏差：只看现在还活着的股票，忽略退市股票。
- 交易成本低估：没有正确考虑手续费、印花税、滑点。
- 成交假设过于理想：默认想买多少就能按理想价格成交。

## 归因分析

归因分析是把回测或模拟盘结果拆开，回答“为什么挣钱、为什么亏钱”。它会按标的、行业/主题、因子、时间窗口、事件窗口、交易成本和回撤片段等维度拆解收益和亏损来源。

归因分析不是给结果找借口，也不能把相关性强行说成因果。它的工程落点是让报告能指出：哪些股票贡献了收益，哪些交易拖累了收益，手续费和滑点吃掉了多少，最大回撤主要发生在什么时间和什么主题上。

## 模拟盘

模拟盘是在真实时间里运行策略，但不发真实订单。它比回测更接近实盘，因为它不能偷看未来数据。

## 实盘

实盘是真实账户、真实资金、真实下单。实盘里程碑最重要的是风控、日志、告警、对账和人工暂停能力。

## OMS

OMS 是订单管理系统，负责管理订单生命周期。它记录订单从创建、提交、成交、撤单到失败的每一步。

## Broker Adapter

Broker Adapter 是券商适配器。不同券商 API 的字段、认证、订单类型都不同，适配器负责把这些差异包装起来，对系统内部提供统一接口。

本项目预计至少有：

- miniQMT 适配器。
- 盈立 OpenAPI 适配器。

## 只读券商接入

只读券商接入是指只调用登录、权限检查、行情查询、账户查询、资金查询、持仓查询、订单查询和成交查询等接口，不向券商提交委托或撤单。这类测试一般不构成真实交易，但会接触账户隐私，仍然需要日志脱敏和访问控制。

只要接口可能在券商侧形成委托、撤销委托、创建条件单、创建预埋单或改变订单状态，就不再是只读接入，必须按真实交易处理。

## 风控

风控是防止系统因为策略错误、数据错误、接口错误或极端行情造成不可接受损失的保护层。

常见风控：

- 单笔订单不能超过指定金额。
- 单只股票不能超过总资产指定比例。
- 一天最多交易多少次。
- 亏损达到阈值后停止交易。
- 禁止买入黑名单股票。
- 非交易时间不允许下单。

## 对账

对账是比较本地记录和券商真实账户状态是否一致。例如本地认为持有 100 股，但券商返回持有 0 股，这就是严重异常。

## 滑点

滑点是预期成交价和实际成交价之间的差异。例如策略看到价格 10.00 想买，但实际成交是 10.03，多出来的 0.03 就是滑点。

## 最大回撤

最大回撤表示策略从历史高点到之后低点的最大跌幅。它衡量的是策略最痛苦的一段亏损。

## 夏普比率

夏普比率用于衡量单位风险带来的收益。它适合辅助比较策略，但不能单独作为判断策略好坏的唯一标准。

## IC

IC 是 Information Coefficient，通常翻译为信息系数。它衡量因子值和未来收益之间的相关性。

简单理解：如果一个因子分数越高，未来收益越高，那么这个因子的 IC 可能为正。

## SHAP

SHAP 是一种解释模型输出的方法。对于一个模型给出的选股评分，SHAP 可以帮助解释哪些特征对这个评分贡献最大。

## LLM

LLM 是大型语言模型，例如 GPT、Claude 等。在本项目里，LLM 更适合做研究助手，而不是交易执行者。

适合 LLM 做的事：

- 阅读公告和财报。
- 生成候选因子代码。
- 帮助解释回测结果。
- 生成策略草案。

不应该让 LLM 做的事：

- 直接决定实盘下单。
- 绕过风控。
- 自动启用未经审核的策略。
