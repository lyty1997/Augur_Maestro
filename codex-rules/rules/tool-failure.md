# 工具失败处理规范

## 基本原则

工具失败后先分析原因，再换方式处理。禁止无脑重复同一失败调用。

## 文件读取失败

- 路径不存在：先确认当前工作目录和文件名。
- 中文乱码：尝试显式 UTF-8。
- 文件过大：先用 `rg`、目录列表或分页方式定位。
- PDF 读取困难：优先查找已提取 Markdown 或文本；若仍不足，再使用 PDF 工具或截图方式定位关键页。

## 命令失败

失败后检查：

- 是否在正确工作目录。
- 是否缺少依赖或虚拟环境。
- 是否受 sandbox、权限或网络限制影响。
- 是否命令本身不适合 PowerShell。

如果失败明显由 sandbox、网络或权限导致，按当前环境要求请求升级权限。不得通过危险替代方式绕过审批。

## 编辑失败

手工编辑使用 `apply_patch`。补丁失败时先重新读取目标片段，确认上下文是否已有用户改动，再生成更小补丁。

不得通过覆盖整个文件来规避冲突，除非用户明确要求并确认风险。

## 测试失败

测试失败时优先定位真实原因。不要为了通过测试而删除安全约束、放宽风控或隐藏异常。
