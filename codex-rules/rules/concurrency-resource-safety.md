# 并发与资源生命周期规范

## 基本原则

所有后台任务、定时器、WebSocket、HTTP 客户端、数据库连接、文件句柄和子进程都必须有明确的创建、使用、取消、关闭和异常处理路径。

## Python 服务

- 不创建无法取消的后台任务。
- `asyncio.create_task` 创建的任务必须被记录、取消或等待，并处理异常。
- 定时任务必须防止重入、并发执行和漂移造成的重复交易意图。
- 应用关闭时必须释放数据库连接、HTTP session、WebSocket 和文件资源。

## 行情与 WebSocket

- 行情订阅必须有订阅、退订、心跳、重连、限频和关闭设计。
- 重连后不得重复注册同一策略或重复触发交易意图。
- 行情断线、延迟、乱序时，必须标记数据状态，不得悄悄使用过期行情做实盘决策。

## 订单与状态机

- 同一交易意图、订单、撤单请求需要幂等键或等价机制。
- 对同一订单的状态更新必须可追踪，避免并发写入导致状态倒退。
- 下单失败和未知状态不得通过自动重试补偿。

## 外部进程

- 启动本地服务、数据脚本或子进程时，需要记录启动方式、端口、工作目录和关闭方式。
- 不遗留无主后台进程。
- 不把真实密钥通过命令行参数暴露。
