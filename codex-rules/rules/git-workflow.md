# Git 工作流规范

## 分支

默认使用 `codex/` 前缀创建工作分支，除非用户指定其他分支策略。

不要在未确认的情况下切换分支、重置工作区或丢弃用户改动。

## 提交

提交前检查：

- `git status`，确认只包含本次任务相关变更。
- 相关测试、格式化、lint 或文档检查已运行，无法运行则说明原因。
- 没有写入密钥、token、真实账户号、身份证明或资金隐私。

提交信息建议使用中文或项目约定格式，简洁说明目的。不要添加 AI 署名、营销式描述或无关正文。

## 禁止事项

- 不执行 `git reset --hard`、`git checkout -- <file>` 等破坏性命令，除非用户明确要求并确认风险。
- 不回滚用户已有改动。
- 不把未确认的真实券商配置、密钥或账户数据加入版本库。
