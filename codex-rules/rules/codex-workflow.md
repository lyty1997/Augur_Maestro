# Codex 工作流规范

## 开始任务

1. 读取 `AGENTS.md`、`docs/README.md`、`codex-rules/global-AGENTS.md` 和本次任务相关规则。
2. 检查相关设计文档和 `docs/00-project/03-open-decisions.md`。
3. 明确任务是否触及架构、交易链路、风控、账户、订单、数据模型、券商适配器或部署方式。
4. 若触及上述范围，先更新 `docs/` 中的设计与待决策问题，并等待用户确认后再写代码。

## 执行原则

- 小步推进，优先完成最小可验证闭环。
- 不把研究、回测、模拟交易、实盘交易混在同一个未经隔离的调用链中。
- 不引入复杂基础设施，除非当前阶段设计文档已说明必要性。
- 不绕过风控、OMS、统一券商网关。
- 不因为测试方便而增加真实交易风险。

## 文档先行

以下变化必须先改设计文档：

- 新增或变更账户、资金、持仓、订单、成交、委托、交易意图等模型。
- 新增或变更 OMS、风控、券商适配器、交易状态机、对账流程。
- 新增真实券商接入、登录、签名、token、WebSocket、行情订阅。
- 新增部署方式、数据库结构、后台任务、定时任务、队列、缓存。
- 新增自动交易能力或任何可能改变券商侧状态的能力。

## 验证与汇报

完成任务后汇报：

- 修改了哪些文件。
- 做了哪些验证。
- 哪些问题仍待确认。
- 是否触碰真实券商接口；默认应为未触碰。

代码改动后按项目现状运行相关检查，例如：

- Python 单元测试：`python -m unittest` 或 `pytest`。
- 已配置时运行格式化、lint、typecheck。
- 前端改动必须启动页面并做实际渲染或截图验证。

若无法运行检查，需要说明原因。
