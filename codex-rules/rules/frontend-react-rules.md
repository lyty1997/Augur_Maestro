# React 前端规范

## 产品定位

Augur_Maestro 前端默认是交易研究和监控工作台，不是营销落地页。界面应服务于扫描、比较、审计、暂停、确认和复盘。

## 安全边界

- M1 阶段前端不得提供真实下单、真实撤单、真实改单按钮。
- 任何未来真实交易按钮必须有权限、环境标识、二次确认、风险提示、审计日志和后端实盘开关。
- UI 不展示真实完整账户号、身份证明、token、密钥或未脱敏资金隐私。

## 交互设计

- 账户、资金、持仓、订单、成交、风控、策略状态应以密集但清晰的运营界面呈现。
- 常见操作使用清晰按钮、图标、筛选器、标签页、表格和状态徽标。
- 不使用花哨装饰替代关键信息。
- 用户可见文案默认简体中文。

## 技术规则

- 优先沿用项目既有 React、TypeScript、组件库和状态管理方式。
- API 调用应通过统一 client 封装，避免页面直接拼 URL 或散落鉴权逻辑。
- 金额、数量、日期时间、订单状态等展示必须有明确格式化。
- 错误状态、空状态、加载状态和权限不足状态都要可见。

## 验证

前端改动后必须实际运行页面并做渲染或截图验证。检查：

- 文本不重叠、不溢出。
- 移动端和桌面端关键视图可用。
- 表格、筛选、弹窗、确认态和错误态可见。
- 没有把危险交易操作暴露到错误环境。
