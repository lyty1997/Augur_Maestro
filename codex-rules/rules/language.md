# 语言与解释规范

## 默认语言

- 与用户对话：中文。
- `docs/` 文档：简体中文。
- 代码注释：中文为主，标准英文术语、协议名、API 名保持原文。
- 用户可见 UI 文案：简体中文。
- 券商申请材料或券商原文引用可按对方要求使用繁体中文或指定格式。

## 解释方式

用户熟悉 Python，但正在学习量化交易。引入新概念时先说明它解决什么问题，再说明工程落点。

例如：

- OMS：解释为订单管理系统，用来接收交易意图、管理订单状态、对接风控和券商。
- 对账：解释为把本地记录与券商真实账户记录核对，发现不一致时暂停并人工确认。
- 滑点：解释为预期成交价与实际成交价差异，会影响回测与实盘表现。

## 不确定性

对不确定的券商 API 行为、交易规则、市场规则，不猜测为事实。应标记为待确认，并说明需要通过官方文档、券商确认、模拟环境或只读查询验证。

## 注释

关键业务规则、交易状态机、风控判断、券商适配器差异、并发和资源生命周期必须写清楚。避免无意义注释。
